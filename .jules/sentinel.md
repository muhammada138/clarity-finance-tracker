## 2024-04-24 - Information Leakage in API Error Responses
**Vulnerability:** The API routes (`backend/app/routes/plaid.py` and `backend/app/routes/insights.py`) caught unhandled exceptions and returned the raw exception string (`str(e)`) in the body of HTTP 500 error responses to the client. This exposed internal details, including full stack traces or third-party service exception messages, to unauthorized users.
**Learning:** Returning unhandled exception messages directly to clients is a common source of sensitive information leakage. The developer should prioritize generic external error messages combined with detailed internal logging to preserve the error context for debugging while obfuscating the underlying mechanism from end users.
**Prevention:** Catch top-level exceptions and replace `detail=str(e)` with a generic `detail="An internal server error occurred"` in HTTP error responses, and use `logger.exception("Contextual message")` to record the full stack trace securely on the server.
